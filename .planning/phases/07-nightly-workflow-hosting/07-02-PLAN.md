<objective>
Land the L-01 Lambda@Edge path-gate carve-out so the OG-meta interceptor passes `/dwca/*` viewer-request URIs straight through to S3 — preserving the existing bot-UA branch for every other path. Without this change, any consumer with a bot UA (Slackbot, Facebook, Twitter, Bluesky, etc.) hitting `https://salishsea.io/dwca/salishsea-occurrences-v1.zip` would receive synthesized OG HTML instead of the binary archive, and Plan 07-03's V-01 smoke check (`Content-Type: application/zip`) would silently break the day a bot UA appeared in the smoke runner.

Purpose: Plan 07-03's first nightly publish MUST be able to fetch the published archive through CloudFront without OG-meta interception. This plan ships that prerequisite as a one-line carve-out + a Jest unit test suite extension, then gates Plan 07-03 on confirmation that the change has been deployed to prod CloudFront.

Output:
- `infra/lib/edge-handler/index.ts` — handler gains a `/dwca/*` early-return at the top (before the existing `isBot(ua)` branch).
- `infra/lib/edge-handler/index.test.ts` — extended with `/dwca/*` carve-out tests proving (a) pass-through for bot UAs, (b) pass-through for non-bot UAs, (c) pass-through with querystrings, (d) all 10 pre-existing tests still pass.
- One commit to `main` that triggers `deploy.yml` to ship the new Lambda@Edge version. Task 2 checkpoint waits for the deploy to finish before declaring the plan done.
</objective>
