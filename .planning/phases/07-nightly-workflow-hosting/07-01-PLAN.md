## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| guard.ts → `dist/dwca/guard-diff.txt` | Untrusted file path (env-overridable in tests) but constant in prod; written only by the workflow's job-private workspace. |
| guard.ts → Supabase Postgres (DSN) | DSN sourced from `SUPABASE_DB_URL` env var; trust-boundary inheritance from Phase 6's `build.ts`. NEVER logged. |
| verify-publish.ts → CloudFront (`https://salishsea.io/dwca/*`) | Public-internet egress; reads our own published artifacts; no auth needed; trust is in the published sha256 sidecar. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-01 | Information Disclosure | guard.ts DSN handling | mitigate | Copy/import Phase 6's `maskDsn` helper; never `console.log(dsn)`; never include DSN in thrown Error message; ATTACH error path scrubs via `maskDsn` before rethrow. Test 5 of guard.test.ts is the unit-level regression gate. |
| T-7-01b | Information Disclosure | guard-diff.txt | accept | The structured diff file contains row counts and byte sizes — no secrets. Workflow uploads it as the failure-issue body (07-03 Plan); operator-readable. |
| T-7-04 | Tampering | verify-publish.ts trust in published sidecar | mitigate by design | The sha256 sidecar is fetched from the SAME origin as the artifact, so an attacker who can swap the artifact can swap the sidecar. V-01 specifically catches **partial uploads** and **cache weirdness**, not malicious tampering — this disposition is **accept** for v1.2 download-only artifact with no signed-checksum requirement. ASVS L1 V6 (cryptography) satisfied — sha256 is FIPS-acceptable; no MD5/SHA-1. |
| T-7-04b | Tampering | parseSha256Sidecar trust in sidecar bytes | mitigate | Validate `[0]` is a 64-char `[0-9a-f]+` string; throw on malformed sidecar; prevents `eval`-like injection if a future caller blindly interpolates the result. Test 1 + a malformed-input case cover this. |

(T-7-02 OIDC overprivilege, T-7-03 peter-evans SHA pin, T-7-05 Lambda@Edge regression are addressed by Plans 07-02 and 07-03 respectively.)
