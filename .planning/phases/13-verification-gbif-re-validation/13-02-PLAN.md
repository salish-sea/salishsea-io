  <behavior>
    - assertIndexeable(result): pure function over a parsed result object.
      - Throws if result.indexeable !== true (message: "GBIF validator: not indexeable").
      - Collects all issues across result.results[].issues where issueCategory ∈ {RESOURCE_INTEGRITY, RESOURCE_STRUCTURE}; throws if any, listing them.
      - Returns a summary object of the non-blocking warnings (METADATA_CONTENT, OCC_INTERPRETATION_BASED, e.g. RESOURCE_CONTACTS_MISSING_OR_INCOMPLETE, COORDINATE_UNCERTAINTY_METERS_INVALID) so main() can report them without failing.
    - submitValidation(zipPath, creds): POST multipart/form-data with the zip as the `file` field + Basic auth header from creds; returns the validation key (UUID) from the response JSON.
    - pollValidation(key, creds): GET the result; loop with ~5–10s interval and a ~5min cap; return the final result once state is no longer RUNNING/QUEUED. Throw on timeout.
    - main(): read GBIF_USER/GBIF_PASS (exit 1 with a clear message + the manual-upload fallback URL if absent); submit dist/dwca/salishsea-occurrences-v1.zip; poll; call assertIndexeable; print PASS with the warning summary, or FAIL with the blocking issues; exit non-zero on failure. On any network non-2xx after retries, print the manual fallback (`https://www.gbif.org/tools/data-validator`) and exit non-zero.
    - NEVER print GBIF_PASS or the Authorization header. (Reuse a maskDsn-style discipline if echoing any URL.)
    - Unit test (validate-gbif.test.ts): exercise assertIndexeable against INLINE fixture JSON — (a) indexeable:true with only METADATA_CONTENT warnings → passes + returns the warnings; (b) indexeable:false → throws; (c) indexeable:true but a RESOURCE_STRUCTURE issue present → throws. NO network.
