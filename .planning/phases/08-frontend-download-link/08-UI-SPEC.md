## In-Section Layout

Source: CONTEXT.md D-03, D-06, D-07, D-09; RESEARCH.md §"New: complete sketch".

### Element order within the new section

1. `<h4>Data download</h4>` — section heading
2. `<p>` — inclusion/exclusion paragraph (see Copywriting below)
3. `<ul class="downloads">` — file rows (see File Rows below)
4. `<p class="freshness">` — relative-time stamp or static fallback (see States below)

### File rows (`<ul class="downloads">`)

Source: CONTEXT.md D-03, D-04; RESEARCH.md §"Alternatives Considered" + open question 1.

Use an unordered list (`<ul class="downloads">`). Each `<li>` holds one primary artifact
and its `.sha256` "verify" link. Two `<li>` elements total:

```
<li>
  <a href="/dwca/salishsea-occurrences-v1.zip" download>salishsea-occurrences-v1.zip</a>
  [size if available]
  <a href="/dwca/salishsea-occurrences-v1.zip.sha256" download class="sha-link">sha256</a>
</li>
<li>
  <a href="/dwca/salishsea-occurrences-v1.parquet" download>salishsea-occurrences-v1.parquet</a>
  [size if available]
  <a href="/dwca/salishsea-occurrences-v1.parquet.sha256" download class="sha-link">sha256</a>
</li>
```

**`.zip` row is first** — DwC-A is the headline/interoperable format. `.parquet` is second.

**Primary links** use the `download` attribute so the browser saves with the canonical
filename instead of rendering or prompting with a path-derived name.

**Size** (e.g., `1.4 MB`): rendered as `<small>` inline after the primary link filename,
separated by a space. Only rendered when `downloadInfo.ok === true` and the byte count
is non-null. Hidden silently when unavailable (no placeholder, no skeleton, no "..." text).

**`.sha256` verify affordance**: rendered as `<a class="sha-link">sha256</a>` inline
after the size (or after the filename if no size). Uses `class="sha-link"` for reduced
visual weight (smaller font-size — see CSS below). No size rendered for `.sha256` files;
they are predictably 65 bytes and that number adds no value.

**Order within a `<li>`:** primary link → size (if present) → sha-link.

## Copywriting Contract

Source: CONTEXT.md D-06, D-07, D-08, D-09; RESEARCH.md §"New: complete sketch".

### Section heading

"Data download"

### Inclusion/exclusion paragraph

The paragraph must:
- Name the format (Darwin Core Archive) with a link to `https://dwc.tdwg.org/`
- Mention the GeoParquet sidecar
- Explicitly list included sources: native SalishSea.io observations + Maplify / Whale Alert (including Orca Network and Cascadia)
- Explicitly list excluded sources: iNaturalist and Happywhale, with the rationale (already published to GBIF by their canonical sources)
- Inline the CC-BY-NC 4.0 license as a linked reference to `https://creativecommons.org/licenses/by-nc/4.0/`

Approved prose (executor MUST use this exact copy or a trivially equivalent rewrite that
preserves all named entities and the rationale clause):

> The full observation dataset is published nightly as a
> [Darwin Core Archive](https://dwc.tdwg.org/), with a GeoParquet sidecar for
> spatial tools. Native SalishSea.io observations and Maplify / Whale Alert sightings
> (including Orca Network and Cascadia) are included; iNaturalist and Happywhale are
> excluded — those are already published to GBIF by their canonical sources. Licensed
> [CC BY-NC 4.0](https://creativecommons.org/licenses/by-nc/4.0/).

### File link labels

| File | Link text |
|------|-----------|
| DwC-A archive | `salishsea-occurrences-v1.zip` |
| GeoParquet sidecar | `salishsea-occurrences-v1.parquet` |
| DwC-A checksum | `sha256` |
| GeoParquet checksum | `sha256` |

No other label text for these links. Sizes render as `<small>` beside the link text,
not as part of the anchor label.

### Size format

Use `formatBytes` helper. Output examples: `65 B`, `812 KB`, `1.4 MB`. Binary
(1 KB = 1024 B) thresholds with decimal display via `Intl.NumberFormat`.

### Last-updated / freshness copy

| State | Copy |
|-------|------|
| In-flight (HEAD not yet resolved) | nothing — render empty string or omit the `<p class="freshness">` content |
| HEAD succeeded, `lastModified` present, gap < 7 days | `updated {relative}` e.g. `updated 6 hours ago`, `updated yesterday`, `updated 3 days ago` — via `Intl.RelativeTimeFormat('en', { numeric: 'auto' })` |
| HEAD succeeded, gap >= 7 days (stale signal) | `updated {YYYY-MM-DD}` — absolute UTC date, e.g. `updated 2026-06-11` |
| HEAD failed (network error / non-2xx) | `Updated nightly at 09:00 UTC.` — static schedule statement |

The 7-day cutoff is the threshold at which relative phrasing ("N days ago") becomes a
soft-alarm that the nightly job has stalled; switching to absolute date makes the staleness
legible without an explicit error surface. Source: CONTEXT.md D-11; RESEARCH.md Pattern 2.

### Primary CTA

No button CTA in this section. Downloads are triggered by clicking the `<a download>`
primary links directly. No "Download" button wrapper.

### Empty state

Not applicable — the section is always rendered (the links are always present). Sizes
and timestamp are simply absent when the HEAD is in-flight or has failed.

### Error state

No error surface rendered to the user. On HEAD failure the section renders with the
static schedule-statement fallback (`Updated nightly at 09:00 UTC.`) and no file sizes.
The failure is captured by the existing Sentry integration without any bespoke
`captureException` call.

### Destructive actions

None. This phase introduces no destructive or confirmation-required actions.

## Interaction & State Behavior

Source: CONTEXT.md D-04, D-12; RESEARCH.md Pattern 1 + Pitfall 5.

### HEAD-on-open

- Fire two parallel HEAD requests (`/dwca/salishsea-occurrences-v1.zip` and
  `/dwca/salishsea-occurrences-v1.parquet`) the **first time** the About dialog opens.
- Triggered inside the existing `onAboutClicked` handler, after calling
  `this.dialogRef.value!.showModal()`.
- Guard: only fire when `this.downloadInfo === null`. Cache the result (both success
  and failure states) for the session. Subsequent opens do not re-fire.
- Use `Promise.allSettled`. Treat any rejection OR non-2xx response as failure.
- On success: populate `zipBytes`, `parquetBytes`, `lastModified` (from `.zip` response).
- On failure: set `downloadInfo = { ok: false }`.

### Dialog usability during HEAD in-flight

The dialog opens immediately and all four download links are clickable before HEAD
resolves. The size text and freshness paragraph are simply absent until the state
resolves. No skeleton, no loading spinner, no disabled state on links.

### Focus & keyboard behavior

The existing dialog already handles focus. No custom focus trap or focus-management
changes needed. The new `<a>` elements inside the dialog are standard anchors — they
receive focus in tab order naturally within the dialog. The `.sha-link` affordances
are standard `<a>` elements and must be keyboard-reachable (do not use `tabindex="-1"`).

The existing close button (`<a class="close-dialog">`) is floated right inside `<h3>`.
The new section does not affect or replicate this.

### Mobile / narrow viewport behavior

Source: existing `@media (max-aspect-ratio: 1)` block in `static styles`.

The `dialog { max-width: 30rem; }` rule applies at all viewport sizes. On portrait-mode
mobile, the dialog modal itself constrains to `30rem` or viewport width, whichever is
smaller. No new `@media` rule is needed for the download section — the existing
block-level flow of `<p>`, `<ul>`, `<li>` wraps naturally within the constraint.

The `.downloads li` items will wrap the sha-link onto a new line if the filename +
size + sha-link exceeds the available width. This is acceptable and expected. Do not
use `white-space: nowrap` on the `<li>`.

## Outbound Link Requirements

Source: RESEARCH.md §Security + existing project convention in `partner-links.test.ts:53`.

All outbound anchors (non-same-origin) must include `target="_blank" rel="noopener noreferrer"`:

| URL | Context |
|-----|---------|
| `https://dwc.tdwg.org/` | Darwin Core explainer in paragraph |
| `https://creativecommons.org/licenses/by-nc/4.0/` | Inline license reference in paragraph |

Primary download links (`/dwca/…`) are same-origin — `target="_blank"` is not required,
but may be used if the executor judges it appropriate for binary download links. The
`download` attribute is required on all four `/dwca/` anchors.
