## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ↔ CloudFront `/dwca/*` | Same-origin (CSP `'self'`). HEAD and GET both pass through Phase 7's L-01 carve-out method-agnostically. No new boundary opened. |
| Browser ↔ `dwc.tdwg.org` | Outbound `<a target="_blank">` navigation — pure anchor, not a fetch. Does NOT consume `connect-src`, `frame-src`, or `script-src`. |
| Browser ↔ `creativecommons.org/licenses/by-nc/4.0/` | Same as above. |
| Lit template ↔ HEAD response values | `Content-Length` numeric and `Last-Modified` string flow through Plan 01 formatters; Lit auto-escapes interpolation; no `unsafeHTML` directive used. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04 | Tampering | `target="_blank"` outbound anchors (reverse-tabnabbing) | mitigate | Per RESEARCH §"Known Threat Patterns" and the project convention enforced by `partner-links.test.ts:53`, BOTH outbound anchors (`dwc.tdwg.org`, `creativecommons.org`) get `target="_blank" rel="noopener noreferrer"`. The DOM test case asserts both `rel` attributes. ASVS V14. |
| T-08-05 | Tampering | Lit template interpolation of HEAD-derived strings | accept (Lit auto-escapes; no `unsafeHTML`) | Lit's `html\`\`` literal escapes string interpolations by default. We do not use `unsafeHTML` / `unsafeStatic` directives anywhere in this plan. ASVS V5. |
| T-08-06 | Denial of Service | HEAD spam on rapid open/close | mitigate | Session cache via `if (this.downloadInfo === null)` guard in `onAboutClicked`. The "HEAD does not refire" test case enforces this behavior. ASVS V14. |
| T-08-07 | Information Disclosure | Surfacing HEAD failures as user-visible errors | mitigate | UI shows the static schedule fallback (`'Updated nightly at 09:00 UTC.'`) — never the underlying error message or status code. Sentry captures the failure via the existing global handler (RESEARCH §"Architectural Responsibility Map"). |
