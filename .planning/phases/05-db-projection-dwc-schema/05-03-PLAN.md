<critical_notes>
**Column contract mirror.** This view MUST emit the exact same 25 columns in the exact same order, with the exact same names and types, as `dwc._native_occurrences` (plan 05-02's `<interface_contract>` block). `UNION ALL` raises at view-creation if any column drifts. Cast widening (e.g., `smallint` → `integer`) was already done on the native side so this side only needs to match.

**Source-distinct audit before encoding the CASE.** POLICY §2.2 Assumption A2 and CONTEXT.md Claude's Discretion both require querying `SELECT DISTINCT source FROM maplify.sightings` against production before finalizing the CASE arms. This plan contains a `[BLOCKING]` audit task (Task 1) that pauses the executor for user-provided list confirmation. The defensive fallback `ELSE 'Whale Alert / Maplify'` covers any unknown source.

**Source filter lever, not source filter activation.** POLICY §4.1 (D-03) explicitly says the per-`maplify.source` drop is "ready, not active" in v1.2 — no source is dropped by default. The filter mechanism lives in `WHERE`; encode it as a commented-out clause inside the WHERE block so future activation is a one-line uncomment.

**Date-precision `eventDate` for Maplify.** POLICY §3.2 says `created_at` is report-receipt time, not sighting time — emitting at second precision would falsely imply sighting-time precision. ALIGN-05 mandates date-only (`YYYY-MM-DD`, no `T`).

**`rwsas` defensive exclusion.** POLICY §5.3 + RESEARCH Open Question 2 default: include `AND source != 'rwsas'` unconditionally. It's a free correctness guard mirroring the ingest filter (`20250919034327_fix_maplify_taxon_mapping.sql:34`).
</critical_notes>
