# Phase 3 — UI Design Contract: Partner Org Hyperlinking

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Note

This phase introduces no new UI surfaces, no new components, and no layout changes. The sole visual change is that partner org names appearing in occurrence body text render as hyperlinks. All design tokens are inherited directly from the existing `obs-summary` Lit component. This contract specifies exactly how the new inline links must look and behave — nothing more.

## Partner Org Link Visual Contract

These are the specific style rules for the new partner org anchor tags rendered inside body text:

```
color: #1976d2          /* matches existing `a` rule in obs-summary.ts */
text-decoration: none   /* matches existing `a` rule */
word-break: break-word  /* matches existing `a` rule */
```

Hover state:
```
color: #1565c0          /* matches existing `a:hover` rule */
```

Attributes on every generated link:
```
target="_blank"
rel="noopener noreferrer"
```

**Source:** REQUIREMENTS.md PARTNER-04, `obs-summary.ts` `a` and `a:hover` rules.

No new CSS rules are needed. The existing `a` selector in `obs-summary.ts` static styles applies automatically to any `<a>` elements rendered inside the shadow root by `unsafeHTML`.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click partner org link | Opens org website in new tab (`target="_blank"`) |
| Hover partner org link | Color changes from #1976d2 to #1565c0 (inherits existing `a:hover` rule) |

No loading state, no tooltip, no confirmation dialog. This is a static inline link.

## Bracket Pattern Contract

Body text containing `[Org Name]` (where "Org Name" matches a CSV entry) must produce:

```
[Org Name](https://example.org)
```

NOT:
```
[[Org Name](https://example.org)]
```

The pre-processing step must detect the `[text]` pattern (no following `(url)`) and replace the bare `[Org Name]` with `[Org Name](url)` before `marked.parse` runs.

**Source:** REQUIREMENTS.md PARTNER-05, CONTEXT.md D-06.

---

## Double-Link Guard Contract

Body text already containing a valid markdown hyperlink matching an org name must not be processed again. Detection: if the pattern `[Org Name](` already exists (case-insensitive on org name), skip injection for that org in that body.

**Source:** REQUIREMENTS.md PARTNER-06, CONTEXT.md D-07.

---

## DOMPurify Configuration Contract

Current call: `domPurify.sanitize(html)` — this strips `target` and `rel` attributes from `<a>` tags by default.

Required update: pass `ALLOWED_ATTR` to preserve partner link attributes:

```ts
domPurify.sanitize(html, {
  ADD_ATTR: ['target', 'rel'],
})
```

This must be applied at the existing sanitize call site in `obs-summary.ts:176` so it covers all rendered body content.

**Source:** REQUIREMENTS.md PARTNER-04 (`target="_blank" rel="noopener noreferrer"`), CONTEXT.md Discretion note on DOMPurify configuration.
