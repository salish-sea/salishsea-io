# Phase 9 Plan 1: Reference Table Foundation Summary

Created the v1.3 provenance graph reference tables — enum type, three tables with RLS, nullable orcid column on contributors, and idempotent seed — verified by a psql assertion snippet that proves SC-1..SC-5 plus the RLS write-closed guarantee.

## What Was Built

### Migration: `supabase/migrations/20260619184037_reference_tables.sql`

Single migration in strict dependency order:

1. **`public.collection_kind` enum** — 5 values: `facebook_group`, `research_dataset`, `acoustic_feed`, `detector`, `direct_app`. `aggregator_ingest` absent by construction (D-09).

2. **`public.providers`** — IDENTITY PK, `slug VARCHAR(50) UNIQUE`, `name VARCHAR(100)`. RLS SELECT policy for anon + authenticated. Seeded: 4 rows.
   - `direct` / SalishSea.io Direct
   - `maplify` / Maplify / conserve.io
   - `inaturalist` / iNaturalist
   - `happywhale` / HappyWhale

3. **`public.organizations`** — IDENTITY PK, `slug VARCHAR(50) UNIQUE`, `name VARCHAR(200)`, `url TEXT NOT NULL`, `rights_holder_text VARCHAR(200) NOT NULL`. RLS SELECT policy. Seeded: 5 rows.
   - `orca-network`, `cascadia`, `tmmc`, `mbari`, `orcasound`

4. **`public.collections`** — IDENTITY PK, `slug VARCHAR(100) UNIQUE`, `name VARCHAR(200)`, nullable `kind public.collection_kind`, nullable `organization_id INTEGER REFERENCES public.organizations(id)`. Btree index on `organization_id`. RLS SELECT policy. Seeded: 21 rows.
   - 10 named collections (kind non-null): `orca-network`, `cascadia`, `tmmc`, `orcasound`, `mbari`, `whale-alert-global`, `whale-alert-alaska`, `inaturalist`, `happywhale`, `salishsea-direct`
   - 11 acronym stubs (kind NULL, organization_id NULL): `psws`, `mcw`, `cww`, `wssji`, `hiws`, `sbw`, `wa`, `ssch`, `sa`, `psww`, `bremerton-fb`

5. **`public.contributors.orcid`** — `ALTER TABLE ... ADD COLUMN IF NOT EXISTS orcid TEXT`. Nullable, no CHECK constraint.
