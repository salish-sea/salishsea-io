    - `writeZip` from `./zip.ts`

    Top-level constants:
    - `const OUT_DIR = 'dist/dwca';`
    - `const OUT_ZIP = path.join(OUT_DIR, 'salishsea-occurrences-v1.zip');`
    - `const OUT_PARQUET = path.join(OUT_DIR, 'salishsea-occurrences-v1.parquet');`
    - `const OUT_OCCURRENCE = path.join(OUT_DIR, 'occurrence.txt');`
    - `const OUT_MULTIMEDIA = path.join(OUT_DIR, 'multimedia.txt');`
    - `const TAB_COLLAPSE_COLS = new Set(['occurrenceRemarks', 'dynamicProperties', 'recordedBy', 'rightsHolder', 'datasetName']);` — per R5

    Helper functions (private to the file):
    - `function buildSelectList(fields: readonly { readonly name: string }[], tabCollapse: ReadonlySet<string>): string` — returns the SELECT projection clause. For names in `tabCollapse`: `regexp_replace("${name}", E'[\\t\\n\\r]+', ' ', 'g') AS "${name}"`. Otherwise: `"${name}"`. Comma-joined.
    - `function describeViewToPgColumns(rows: Record<string, unknown>[]): PgColumn[]` — converts DESCRIBE result rows to `PgColumn[]`. DESCRIBE returns objects with `column_name` keys; map and use the array index +1 as ordinal.
    - `function maskDsn(s: string): string` — returns `'<redacted>'` if the string contains `'://'`; used in error logging to never echo the DSN.

    The single `main()` async function (no top-level await; tests can import without running):

    1. Read DSN: `const dsn = process.env.SUPABASE_DB_URL;` — if falsy, `console.error('SUPABASE_DB_URL is not set')` and `process.exit(1)`. Never include the DSN in any error message.

    2. Create output directory: `await fs.mkdir(OUT_DIR, { recursive: true });`.

    3. Create DuckDB instance and connection: `const db = await DuckDBInstance.create(':memory:');` `const conn = await db.connect();`. Wrap the rest of `main()` in `try { ... } finally { conn.closeSync(); }` so the DuckDB process always closes.

    4. Install + load extensions: `await conn.run('INSTALL postgres; LOAD postgres;');` then `await conn.run('INSTALL spatial; LOAD spatial;');`.

    5. ATTACH: `await conn.run(`ATTACH '${dsn}' AS pgdb (TYPE postgres, READ_ONLY)`);`. Wrap in try/catch; on error, log a sanitized message (`'Failed to attach Postgres: ' + maskDsn(String(err))`) and rethrow.

    6. F-02 assertion for `dwc.occurrences`:
       - `const occDescribeReader = await conn.runAndReadAll('DESCRIBE pgdb.dwc.occurrences');`
       - `const occPgCols = describeViewToPgColumns(occDescribeReader.getRowObjects());`
       - `assertFieldAlignment(occPgCols, OCCURRENCE_FIELDS, 'occurrences');` — throws AlignmentError on drift; the outer try/catch around main() reports it and exits 1.

    7. F-02 assertion for `dwc.multimedia`:
       - Same pattern with `'DESCRIBE pgdb.dwc.multimedia'` and `MULTIMEDIA_FIELDS`, `'multimedia'`.

    8. Pre-COPY row-count guard for `dwc.occurrences`: `const occCount = await assertNonZeroRows(conn, 'pgdb.dwc.occurrences');`. The multimedia view may legitimately be empty in some local DB states; do NOT block on its row count.

    9. COPY occurrence.txt: build the SELECT list with `buildSelectList(OCCURRENCE_FIELDS, TAB_COLLAPSE_COLS)` and run:
       `COPY (SELECT ${selectList} FROM pgdb.dwc.occurrences) TO '${OUT_OCCURRENCE}' (FORMAT csv, DELIMITER '\t', HEADER true, QUOTE '', ESCAPE '', NULLSTR '', ENCODING 'UTF-8')`.
       Note the path uses absolute string interpolation; OUT_OCCURRENCE is relative (`dist/dwca/occurrence.txt`) which DuckDB resolves against the CWD. Build.ts is invoked from the repo root by `npm run build:dwca`, so CWD is reliable.

    10. COPY multimedia.txt: similar, with `buildSelectList(MULTIMEDIA_FIELDS, TAB_COLLAPSE_COLS)` (only `rightsHolder` from MM is in the collapse set; the others are URIs or constants, no tab risk).

    11. COPY parquet:
        - Build the SELECT list using all 25 occurrence field names PLUS the geometry expression. Use `ST_Point("decimalLongitude", "decimalLatitude") AS geometry` — NOT `ST_AsWKB(...)`. This is the R1 override of CONTEXT.md G-01.
        - The occurrence column list uses the plain `"${name}"` form (no tab collapse needed in parquet because there is no delimiter to escape).
        - Run: `COPY (SELECT ${plainOccList}, ST_Point("decimalLongitude", "decimalLatitude") AS geometry FROM pgdb.dwc.occurrences) TO '${OUT_PARQUET}' (FORMAT parquet, COMPRESSION snappy)`.

    12. R1 empirical verification — query parquet kv_metadata BEFORE declaring success:
        - `const geoReader = await conn.runAndReadAll(`SELECT key::text AS k, value::text AS v FROM parquet_kv_metadata('${OUT_PARQUET}') WHERE key = 'geo'::blob`);`
        - `const geoRows = geoReader.getRowObjects();`
        - If `geoRows.length === 0`: `throw new Error('R1 FAILURE: parquet has no "geo" metadata after COPY. The ST_Point GEOMETRY auto-emit assumption is wrong for this DuckDB version. Escalation: either upgrade DuckDB or use the KV_METADATA injection fallback (RESEARCH §R1 option B).')`.
        - Parse `geoRows[0].v` as JSON. Assert: `parsed.version === '1.0.0'`, `parsed.primary_column === 'geometry'`, `parsed.columns?.geometry?.encoding === 'WKB'`. If any assertion fails, throw with the parsed JSON in the error message (no DSN in it).
        - Log success: `console.log(`[build:dwca] GeoParquet geo metadata: version=${parsed.version}, primary_column=${parsed.primary_column}, encoding=${parsed.columns.geometry.encoding}`);`.

    13. Row-count parity sanity check (defense in depth for DWCA-06):
        - `const parquetCountReader = await conn.runAndReadAll(`SELECT COUNT(*) AS n FROM read_parquet('${OUT_PARQUET}')`);`
        - `const parquetCount = parquetCountReader.getRowObjects()[0]!.n as bigint;`
        - Assert `parquetCount === occCount` (both bigints). Throw if not equal: `throw new Error(`Parquet row count mismatch: parquet=${parquetCount}, view=${occCount}`)`.

    14. Zero-byte guard: `await assertNoZeroByteFile(OUT_OCCURRENCE); await assertNoZeroByteFile(OUT_MULTIMEDIA); await assertNoZeroByteFile(OUT_PARQUET);` — multimedia.txt may have only a header line if the view is empty, but the header alone is >0 bytes (this is intentional per CONTEXT.md F-05 — HEADER true is always set).

    15. Compute temporal coverage: `const tempReader = await conn.runAndReadAll(`SELECT MIN("eventDate") AS begin, MAX("eventDate") AS end FROM pgdb.dwc.occurrences`);` `const tempRow = tempReader.getRowObjects()[0]!;` `const temporalCoverage = { begin: String(tempRow.begin).slice(0, 10), end: String(tempRow.end).slice(0, 10) };`. The slice(0, 10) truncates `2026-06-17T12:34:56Z` → `2026-06-17` so EML carries the calendarDate form per POLICY §6.5.

    16. Read `dwc.datasets` (one row in v1.2 per POLICY §6.2 D-16): `const datasetsReader = await conn.runAndReadAll('SELECT * FROM pgdb.dwc.datasets LIMIT 1');` `const datasetsRow = datasetsReader.getRowObjects()[0] as unknown as DatasetsRow;`. If `datasetsReader.getRowObjects().length === 0`: throw `'dwc.datasets is empty — expected exactly 1 row per POLICY §6.2 D-16'`.

    17. Build XML strings: `const metaXml = buildMetaXml(OCCURRENCE_FIELDS, MULTIMEDIA_FIELDS);` `const emlXml = buildEml({ datasets: datasetsRow, temporalCoverage });`.

    18. Read the two .txt files into buffers: `const occBuf = await fs.readFile(OUT_OCCURRENCE);` `const mmBuf = await fs.readFile(OUT_MULTIMEDIA);`.

    19. UTF-8 BOM check before zipping (defense in depth for DWCA-04): if `occBuf[0] === 0xEF || mmBuf[0] === 0xEF`: throw `'BOM detected in DuckDB output — expected UTF-8 no BOM'`. (DuckDB never writes BOM per RESEARCH §T2 — this is a sanity check that surfaces if a future DuckDB version regresses.)

    20. Assemble zip: `await writeZip(OUT_ZIP, [{ name: 'meta.xml', content: Buffer.from(metaXml, 'utf8') }, { name: 'eml.xml', content: Buffer.from(emlXml, 'utf8') }, { name: 'occurrence.txt', content: occBuf }, { name: 'multimedia.txt', content: mmBuf }]);` — entry order is meta → eml → occurrence → multimedia (DwC-A convention).

    21. Final zero-byte guard on the zip: `await assertNoZeroByteFile(OUT_ZIP);`.

    22. Success log: `console.log(`[build:dwca] OK — ${occCount} occurrence rows, parquet=${OUT_PARQUET}, zip=${OUT_ZIP}`);`.
