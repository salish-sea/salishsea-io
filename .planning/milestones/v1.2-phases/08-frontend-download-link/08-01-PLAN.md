## Artifacts this phase produces (Plan 01 slice)

### `src/download-info.ts`
- `export type DownloadInfo = { ok: true; zipBytes: number | null; parquetBytes: number | null; lastModified: string | null } | { ok: false }`
- `export function formatBytes(bytes: number): string` — binary thresholds (1 KB = 1024 B), decimal display, units `B`, `KB`, `MB`, `GB`. Singular `B` for `bytes < 1024`; `Intl.NumberFormat` with `maximumFractionDigits: 1` for the others.
- `export function formatRelativeTime(lastModifiedHeader: string, now?: Temporal.Instant, absoluteAfterDays?: number): string` — buckets: minutes < 60 → `updated N minutes ago` (clamped ≥ 1), hours < 24 → `updated N hours ago`, days < 7 → `updated N days ago`, `>= 7` days → `updated YYYY-MM-DD` (UTC `PlainDate.toString()`). Returns `''` when `Date.parse(header)` is `NaN`. Uses `Intl.RelativeTimeFormat('en', { numeric: 'auto' })`.
- `export async function fetchArchiveMetadata(basePath?: string): Promise<DownloadInfo>` — fires `Promise.allSettled` of two HEADs (`${base}.zip` + `${base}.parquet`), where `base` defaults to `/dwca/salishsea-occurrences-v1`. Returns `{ ok: false }` if either result is rejected OR `!response.ok`. On success, returns `{ ok: true, zipBytes, parquetBytes, lastModified }` where `zipBytes`/`parquetBytes` come from `Number(headers.get('content-length')) || null` and `lastModified` comes from the `.zip` response's `last-modified` header (may be `null`).

### `src/download-info.test.ts`
Vitest specs (no `// @vitest-environment jsdom` pragma — these are pure-function tests; jsdom is unnecessary for the helper-only file and would only slow the run):
- `formatBytes: 0 → '0 B'`
- `formatBytes: 65 → '65 B'` (the predictable .sha256 sidecar size)
- `formatBytes: 1024 → '1 KB'` (boundary)
- `formatBytes: 1536 → '1.5 KB'`
- `formatBytes: 1024 * 1024 → '1 MB'` (boundary)
- `formatBytes: 1.4 * 1024 * 1024 → '1.4 MB'`

- `formatRelativeTime: 2 days ago → 'updated 2 days ago'`
- `formatRelativeTime: exactly 7 days ago → absolute 'updated YYYY-MM-DD'`
- `formatRelativeTime: 14 days ago → absolute 'updated YYYY-MM-DD'`
- `formatRelativeTime: malformed header → ''`
- `fetchArchiveMetadata: both HEADs succeed → { ok: true, zipBytes, parquetBytes, lastModified } sourced from .zip headers`
- `fetchArchiveMetadata: one HEAD rejects → { ok: false }`
- `fetchArchiveMetadata: one HEAD returns 503 (response.ok === false) → { ok: false }`
- `fetchArchiveMetadata: success with missing Content-Length on either response → corresponding {zip|parquet}Bytes is null, info.ok stays true`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ↔ CloudFront `/dwca/*` | Same-origin HEAD/GET. Response headers (`Content-Length`, `Last-Modified`) are CloudFront-controlled but not user-controlled. |
| Browser ↔ external CDN URLs | Plan 01 has no outbound anchors — those land in Plan 02. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | `formatBytes` (`Content-Length` numeric coercion) | mitigate | `Number(headers.get('content-length')) \|\| null` — falsy/NaN/missing collapses to `null` (the "size unavailable" sentinel). Hide the size silently in Plan 02 per CONTEXT D-12. Test case asserts the missing-header path returns `null` in the `DownloadInfo`. |
| T-08-02 | Tampering | `formatRelativeTime` (RFC 7231 `Last-Modified` parsing) | mitigate | Tolerant `Date.parse()` + `Number.isNaN` short-circuit returns `''`. Lit auto-escapes the resulting string interpolation. Test case covers a malformed header. |
| T-08-03 | Denial of Service | `fetchArchiveMetadata` retry storm | accept (per CONTEXT D-04 / RESEARCH Pitfall 5) | One fetch per session in the caller (Plan 02 wires session cache). Plan 01 does not retry; Sentry surfaces failures. ASVS V14 — no widening of `connect-src` needed (same-origin per RESEARCH Pitfall 3). |
| T-08-SC | Tampering | npm installs | n/a | No new packages — RESEARCH §"No `npm install` step". The Package Legitimacy Audit table in RESEARCH §"Package Legitimacy Audit" is empty by design. |

**ASVS coverage (RESEARCH §"Security Domain"):**
- V5 Input Validation: `Number(...) || null` on `Content-Length`; `Number.isNaN(Date.parse(...))` short-circuit on `Last-Modified`. Both implemented in this plan.
- V14 Configuration: CSP unchanged (CONTEXT D-13). Same-origin HEAD is allowed by the existing `connect-src 'self'`. No edit to `index.html`.
</threat_model>
