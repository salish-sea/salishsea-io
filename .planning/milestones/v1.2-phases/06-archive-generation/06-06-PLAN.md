As an alternative, use `(HAS_DSN ? test : test.skip)('name', ...)`.

    - `execSync('npm run build:dwca', { stdio: 'inherit', env: { ...process.env, SUPABASE_DB_URL: DSN } });` — runs the actual build. This may take ~10-30s. If the build fails, beforeAll throws and every test in the file fails clearly.

    (1) **DWCA-01 — zip exists with 4 entries**:
    - Assert `existsSync(ZIP)` and `statSync(ZIP).size > 0`.
    - Use `yazl` or a simple ZIP central directory parser? Simpler: shell out via `execSync('unzip -l ' + ZIP, { encoding: 'utf8' })` and assert the output contains the four expected names. (`unzip` is available on macOS by default and on the ubuntu GH runner.) Verify each of `meta.xml`, `eml.xml`, `occurrence.txt`, `multimedia.txt` appears as a line.

    (2) **DWCA-01 secondary — parquet exists, non-empty**:
    - Assert `existsSync(PARQUET)` and `statSync(PARQUET).size > 0`.

    (3) **DWCA-02 — meta.xml field indices align with occurrence.txt header columns**:
    - Extract `meta.xml` from the zip (use `execSync('unzip -p ' + ZIP + ' meta.xml', { encoding: 'utf8' })`).
    - Regex-extract `<field index="(\d+)" term="([^"]+)"/>` from the core block (text between `<core ...>` and `</core>`). Sort by index. The resulting `(index, term)` list must equal `OCCURRENCE_FIELDS.map((f, i) => [String(i), f.termUri])`.
    - Repeat for the `<extension>` block: the (index, term) list must equal `MULTIMEDIA_FIELDS.map((f, i) => [String(i), f.termUri])`.
    - The header row of `occurrence.txt` (line 0, split on `\t`) must equal `OCCURRENCE_FIELDS.map(f => f.name)`.
    - The header row of `multimedia.txt` must equal `MULTIMEDIA_FIELDS.map(f => f.name)`.

    (4) **DWCA-03 — multimedia anti-join is empty**:
    - Read `occurrence.txt`. Skip the header line. Build `occIds = new Set(lines.map(l => l.split('\t')[0]))`.
    - Read `multimedia.txt`. Skip the header line. For each non-empty row, the coreId (column 0) must be in `occIds`.
    - If `multimedia.txt` has only the header (no data rows), the test still passes (anti-join of an empty set against any set is empty). Document this in a comment.

    (5) **DWCA-04 — no BOM on either CSV**:
    - `readFileSync(OCC_TXT)` (returns Buffer); assert `buf[0] !== 0xEF || buf[1] !== 0xBB || buf[2] !== 0xBF` (the BOM sequence). Same for `MM_TXT`.

    (6) **DWCA-04 secondary — embedded tabs/newlines collapsed in freetext columns**:
    - For each line in `occurrence.txt` (post-header), assert `line.split('\t').length === OCCURRENCE_FIELDS.length` (exactly 25 fields per row). If any freetext column carried an unescaped tab, the row would have >25 fields. This is the primary DWCA-04 invariant check.

    (7) **DWCA-06 — parquet GeoParquet 1.0.0 metadata + row count parity**:
    - Open a fresh DuckDB instance in the test: `const db = await DuckDBInstance.create(':memory:'); const conn = await db.connect(); await conn.run('INSTALL spatial; LOAD spatial;');`.
    - Query: `SELECT key::text AS k, value::text AS v FROM parquet_kv_metadata('${PARQUET}') WHERE key='geo'::blob`. Assert exactly one row. Parse the JSON in `v`. Assert: `version === '1.0.0'`, `primary_column === 'geometry'`, `columns.geometry.encoding === 'WKB'`. If `columns.geometry.geometry_types` is present (it is in DuckDB 1.1+), assert it includes `'Point'` or contains a string starting with `'Point'`.
    - Column count: `DESCRIBE SELECT * FROM read_parquet('${PARQUET}')` row count must equal `26` (25 dwc cols + geometry).
    - Round-trip geometry: `SELECT ST_AsText(geometry) AS wkt FROM read_parquet('${PARQUET}') LIMIT 5` — every `wkt` value must start with `'POINT('` (case-insensitive).
    - Row-count parity vs the source view: this requires an ATTACH inside the test. To keep the test self-contained, attach Postgres again: `await conn.run("ATTACH '" + DSN + "' AS pgdb (TYPE postgres, READ_ONLY);")`. `SELECT COUNT(*) FROM pgdb.dwc.occurrences` and `SELECT COUNT(*) FROM read_parquet('${PARQUET}')` must be equal.
    - `conn.closeSync()` after the test.

    (8) **DWCA-02 round-trip — known seed record column-to-term mapping** (RESEARCH §T8 (a) hand-wave but make it concrete):
    - Pick the first data row of `occurrence.txt`. For each of the 25 fields, log the column name (`OCCURRENCE_FIELDS[i].name`) and the value. The test passes as long as the row has exactly 25 fields and the columns at indices 19 and 22 are interpretable strings (rightsHolder name + license URI). No deeper semantic check — that is the GBIF validator's job (Task 2).
