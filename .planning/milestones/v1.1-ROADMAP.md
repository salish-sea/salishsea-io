**Status:** ✅ SHIPPED 2026-04-18

## Overview

Partner organization names appearing in occurrence body text are automatically hyperlinked to their websites. Maintained via a CSV file (name, url columns) editable without touching TypeScript. DOMPurify configured to preserve target and rel attributes so links open in new tabs.

## Milestone Summary

**Key Decisions:**

- Vite `?raw` import for `partners.csv` — no build plugin, no runtime fetch, parsed once at module load
- Single-pass combined regex per org handles bracket pattern and plain name without double-substitution
- Longest-name-first sort prevents short names (NOAA) matching inside longer ones (NOAA Fisheries)
- Link text always uses canonical CSV name regardless of match casing (brand capitalization preserved)
- Custom marked renderer on `injectPartnerLinks` call site to add target/rel — not a global renderer override
- DOMPurify `ADD_ATTR: ['target', 'rel']` paired with `rel="noopener noreferrer"` (tab-napping mitigation)

**Issues Resolved:**

- WR-01: Pre-existing `target="_new"` on attribution/photo links corrected to `target="_blank"`
- WR-02: `onDelete` async error now caught and logged instead of producing unhandled rejection
- WR-03: Partner regex split on existing markdown links prevents matching names inside URLs
